1. Carriles (lanes) y notación

Cada nodo del workflow pertenece a uno de 6 carriles que representan al actor/sistema responsable. Los nodos están coloreados por carril en todos los diagramas.

👤 Técnico 📱 Frontend PWA 🔧 Backend módulo ⏰ Saga / Outbox 🏢 ERP on-prem 💾 Azure Blob
CarrilResponsabilidadEjemplos
👤 TécnicoDecisiones humanas, captura UXTap en equipo, llenar wizard, firmar
📱 Frontend PWAHTTP síncrono, cache local, validación UIGET /equipos, lectura EquipoLocal
🔧 Backend móduloHandlers, aggregates Marten, invariantesHandler IniciarInspeccion, V-F1..V-F7
⏰ Saga / OutboxAsync Wolverine (retry, dead-letter, idempotency)CerrarInspeccionSaga, EjecutarOTSaga
🏢 ERP on-premSQL Server relacional (preop, MYE, inventario)Recibe REST sobre VPN (ADR-001)
💾 Azure BlobStorage propio del módulo (adjuntos)SAS upload directo, soft delete

PathOrigenDestinoAcción
Bloqueado por I-I2 (sin rutina)N8End 2AContactar admin del catálogo
Inspección activa para el equipo (I-I1)N10End 2BReabre la existente
V-F1..V-F7 fallanN21N19Volver a corregir
OT 4xx perm o agotó retryN66N72 → RetryReintento manual desde panel admin
OT rechazada por aprobadorN42N50Cierre sin OT con razón
Cancelación: el técnico puede ejecutar CancelarInspeccion mientras el aggregate está EnEjecucion. Emite InspeccionCancelada_v1 y termina en estado terminal Cancelada. NO contacta MYE. No se muestra en el diagrama para no saturar.
